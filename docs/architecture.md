# Conbook 구조 문서

이 문서는 지금 프로젝트가 무엇을 만들고 있는지 쉽게 설명하기 위한 문서입니다. 전문적인 말은 줄이고, 팀원이 바로 역할을 이해할 수 있게 적었습니다.

## 우리가 만들 것

이 프로젝트는 크게 3가지를 합니다.

1. 대화 내용을 저장한다.
2. 대화에서 insight를 뽑는다.
3. 그 결과를 화면에 보여준다.

즉, 목표는 대화가 쌓이는 곳, insight가 만들어지는 곳, 시각화되는 곳을 각각 구현하는 것입니다.

## 각 부분의 역할

- 대화하는 곳: 사용자의 대화 내용이 들어오는 원본 데이터입니다.
- 대화 내용을 저장하는 곳: 대화 내용을 JSON 형태로 모아두는 파일입니다.
- insight를 추출하는 곳: 대화에서 주제, 관심사, 흐름을 정리하는 단계입니다.
- 시각화하는 곳: 정리된 결과를 사람이 읽기 쉽게 보여주는 화면입니다.

## 지금 임시로 만든 주요 파일 역할

- [main.ipynb](main.ipynb): 대화 내용을 읽고, insight를 뽑는 작업을 실험하는 곳입니다.
- [streamlit_app.py](streamlit_app.py): 저장된 결과를 화면에 보여주는 곳입니다.
- [result4.json](result4.json), [result5.json](result5.json): 시각화에 쓰는 결과 예시입니다.
- [a_messages.json](a_messages.json), [a_messages2.json](a_messages2.json): 대화 내용 예시입니다.
- [test_data/](test_data/): 저장 방식이나 결과를 확인할 때 참고하는 예시 데이터입니다.

## 저장되는 데이터 모양

현재 결과 JSON은 대략 아래처럼 생겼습니다.

```json
{
  "ok": true,
  "summary": {
    "summary": "전체 대화 요약",
    "interests": ["관심사1", "관심사2"],
    "tags": [],
    "direction": [],
    "timestamp": "..."
  },
  "interests": [
    {
      "interest": {
        "name": "관심사 이름",
        "tags": ["태그1", "태그2"],
        "direction": [],
        "timestamp": "..."
      },
      "direction": {
        "summary": "관심사별 흐름 요약",
        "interest_name": "관심사 이름",
        "direction": [
          {
            "entity": "발화 식별자",
            "description": "대화 흐름 설명"
          }
        ],
        "tags": [],
        "timestamp": "..."
      }
    }
  ]
}
```

핵심은 아래 3단계입니다.

- `summary`: 대화 전체를 한 줄로 요약합니다.
- `interests`: 대화에서 드러난 관심사 목록을 담습니다.
- `direction`: 관심사별로 관련 발화와 설명을 묶어 흐름을 보여줍니다.

## 지금 대충 화면에서 보여주는 방식

[streamlit_app.py](streamlit_app.py)는 이 JSON을 다음처럼 렌더링합니다.

- 사이드바에서 데이터 소스를 선택합니다.
- 기본 파일, 업로드, 직접 입력 중 하나로 JSON을 불러옵니다.
- 전체 요약과 추출된 관심사를 먼저 보여줍니다.
- 관심사별 항목은 접어서 볼 수 있게 보여주고, 태그와 흐름 설명을 확인할 수 있습니다.
- 원본 JSON도 함께 확인할 수 있습니다.

## 팀 작업 기준

- 파일 저장 담당은 대화 원본과 결과 JSON이 같은 흐름으로 저장되도록 맞추면 됩니다.
- 챗봇과의 대화 담당은 파일 저장 영역과 매끄럽게 데이터가 이어질 수 있도록 맞추면 됩니다.
- 시각화 담당은 저장된 JSON을 읽어서 화면에 보여주면 됩니다.
- 새 화면을 만들 때는 전체 요약, 관심사 목록, 관심사별 흐름만 먼저 맞추면 됩니다.
